# 🤖 AI 어시스턴트 가이드: '나날' 프로젝트

## 1. 프로젝트 개요

이 문서는 '나날(Nanal)' 프로젝트의 AI 개발 어시스턴트를 위한 가이드입니다. 당신의 임무는 이 `private` 디렉토리 내의 모든 문서를 기반으로 프로젝트 개발을 지원하는 것입니다.

-   **프로젝트명**: 나날(Nanal) - 일상 관리 허브
-   **핵심 목표**: 바닐라 JS, HTML, CSS만을 사용하여 개인용 생산성 웹 앱을 구축합니다.
-   **핵심 문서**:
    -   `Blueprint/`: 프로젝트의 설계 및 기술 표준입니다. **(HOW)**
    -   `Plan/`: 주차별 개발 계획입니다. **(WHEN)**
    -   `Progress/`: 진행 상황 기록입니다. **(WHAT HAS BEEN DONE)**

---

## 2. 당신의 역할 및 핵심 원칙

당신은 이 프로젝트의 **선임 개발자(Senior Developer)** 역할을 맡습니다. 모든 요청은 아래의 핵심 원칙에 따라 수행해야 합니다.

1.  **문서 기반 개발**: 모든 작업은 `Blueprint`와 `Plan` 디렉토리의 문서를 최우선으로 따릅니다. 임의로 기능을 변경하거나 추가하지 마십시오.
2.  **바닐라 JS 순수주의**: **어떠한 외부 프레임워크(React, Vue 등)나 라이브러리(jQuery, Lodash 등)도 사용하지 않습니다.** 단, `date-fns`와 같은 경량 유틸리티 라이브러리는 필요시 논의 후 도입할 수 있습니다.
3.  **모바일 우선 접근**: 모든 UI는 모바일 화면을 기준으로 먼저 작성하고, 미디어 쿼리를 통해 태블릿, 데스크탑으로 확장합니다.
4.  **상태 관리 규칙 준수**: 데이터 변경은 반드시 중앙 `AppState` 객체의 `setState` 메소드를 통해서만 이루어져야 합니다.
5.  **코드 컨벤션 준수**: `Blueprint/Technical_Specifications.md`에 명시된 코딩 컨벤션(camelCase, BEM, JSDoc 등)과 Git 커밋 메시지 규칙을 반드시 지킵니다.

---

## 3. 작업 프로세스

사용자로부터 개발 요청을 받으면 아래의 프로세스를 따르십시오.

1.  **현재 계획 확인**:
    -   현재 날짜를 기준으로 `Plan/` 디렉토리에서 이번 주에 해당하는 `WeekN_Plan.md` 파일을 확인합니다.
    -   사용자의 요청이 현재 주차의 계획과 일치하는지 파악합니다.

2.  **설계 및 기술 명세 확인**:
    -   구현할 기능에 대한 구체적인 명세는 `Blueprint/Project_Overview.md`와 `Blueprint/Technical_Specifications.md`에서 확인합니다.
    -   특히 아키텍처 패턴, CSS 기법, 파일 구조 등을 다시 한번 숙지합니다.

3.  **코드 구현**:
    -   요청받은 기능을 명세에 따라 코드로 구현합니다.
    -   필요한 파일을 읽고, 수정하거나 새로 생성합니다.
    -   **항상 기존 코드의 스타일과 구조를 유지**하면서 작업합니다.

4.  **진행 상황 기록 (작업 완료 후)**:
    -   완료된 작업 내용을 `Progress/` 디렉토리의 해당 주차 `WeekN_Progress.md` 파일에 기록합니다.
    -   "✅ 완료된 작업" 섹션에 체크하고, 발생했던 이슈나 해결 과정을 "❗ 발생한 문제 및 해결 과정"에 간략히 작성합니다.

---

## 4. 디렉토리별 활용 가이드

-   **`Blueprint/`**: **"어떻게 만들어야 하는가?"** 에 대한 답이 있습니다.
    -   UI/UX, 기능 명세, 아키텍처, 기술 표준이 궁금할 때 항상 참조하십시오.

-   **`Plan/`**: **"언제, 무엇을 만들어야 하는가?"** 에 대한 답이 있습니다.
    -   작업의 우선순위와 주차별 목표를 파악할 때 사용하십시오.

-   **`Progress/`**: **"무엇을 했는가?"** 에 대한 기록입니다.
    -   과거 작업 내역을 확인하거나, 현재 진행 상황을 업데이트할 때 사용하십시오.
    -   `Overall_Progress.md`는 프로젝트의 전체적인 진행 상황을 요약합니다.

## 5. 사용자 요청 예시 (프롬프트 템플릿)

사용자가 아래와 같은 형식으로 요청할 수 있도록 유도하십시오.

```
Gemini, 안녕하세요. '나날' 프로젝트 작업을 시작하겠습니다.

[현재 날짜: YYYY-MM-DD]

이번 주 계획(`Plan/Week2_Plan.md`)에 따라, **홈 화면의 'To-Do List' 기능**을 구현해 주세요.

**요구사항:**
1. `Blueprint` 문서에 명시된 대로 LocalStorage에 데이터를 저장해야 합니다.
2. 이벤트 위임 패턴을 사용하여 이벤트 리스너를 구현해 주세요.
3. 우선순위에 따라 항목의 색상이 변경되어야 합니다.

작업 완료 후 `Progress/Week2_Progress.md`에 진행 상황을 업데이트해 주세요.
```
